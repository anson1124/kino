Message wire format

FRAME				CONTENT							        DESCRIPTION
(of n total frames)	
=====================================================================================
0					Receiver socket ID				        Identity of the receiving socket
1					[0]                                     Empty frame
..
                    .... more ......
                    ROUTER URI
                    ROUTER ID                               Message routing
..
                    .... more ......
                    Callback Version
                    Callback Message ID                     Callbacks
..
n-16                Message Hops                            Number of times a Message was sent away to another node
n-15				Message Routing Entries Count           Number of message route entries
                                                                Own Socket Identities and URIs added by every MessageRouter, forwarding the message
                                                                Each entry takes 2 frames: 1 - URI, 2 - socket ID
n-14				Routing Start Frame Offset              Offset of the first message route frame
n-13                TraceOptions                            Type of tracing to record for the message
n-12				Version							        Message Version
n-11			    Message ID						        Message Identifier
n-10				Receiver ID						        Socket Identity of the receiver of the message. Empty frame, for the non-specific receiver
n-9					Distribution pattern			        One of the following distribution patterns {Unicast, Broadcast}
n-8					Correlation ID					        Unique identifier, copied onto all messages, spawned within the same flow, from the initial one
n-7                 Callback Entries Count                  Number of callback entries:
                                                                Frames: 1 - Callback Message Identifier, 2 - Callback Version
n-6                 Callback Start Frame Offset             Offset of the first callback messages frames
n-5					Callback Receiver ID			        Value from this field will be copied to Receiver Identity when 
                                                                Message Identifier becomes equal to Callback Message Identifier
n-4					TTL								        Message Time-to-Live in msec.
n-3                 Wire Format Version                     Version of the message wire format
n-2					[0]                                     Empty frame
n-1					Body							        Message content

ActorHost is responsible for:
- setting of Receiver ID to the value of Callback Receiver ID, when for the current message 
    Message ID == Callback Message ID and Version == Callback Version
- propagation of Correlation ID from incoming messages to all outgoing messages

Message handler is selected based on the following algorithm:
if 
	ReceiverIdentifier is set, 
then
	Version && Receiver ID
else
	Version && Message ID
